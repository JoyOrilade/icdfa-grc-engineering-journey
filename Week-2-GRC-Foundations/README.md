## Key Topics & Learning Outcomes

- **Regulatory Comparisons**:
  - GDPR: Applies to EU-based orgs, non-EU targeting EU individuals; key principles (lawfulness, data minimization, accountability); fines up to €20M or 4% global turnover.
  - HIPAA: Covers US healthcare entities and associates; rules for privacy, security, breach notification; tiered penalties up to $50,000 per violation, $1.5M annual max.
  - PCI-DSS: For payment card handlers; 12 requirements across 6 objectives (e.g., secure networks, protect data, vulnerability management).
  - ISO 27001/27002: ISMS framework (27001) complemented by control guidelines (27002) for implementation.

- **Industry Compliance**:
  - Healthcare: HIPAA rules for ePHI protection.
  - Finance/Retail: PCI-DSS for secure card data handling.
  - Tech/Education: ISO standards and GDPR for data integrity/confidentiality.

- **Case Study Insights**:
  - Challenges: Overlapping regulations, cross-border transfers (EU-US privacy issues).
  - Strategies: Unified compliance programs, gap remediation, documentation, training, auditing.

- **Practical Skills**:
  - DPIA Template: Steps for risk assessment (e.g., identify risks, consult stakeholders, assess proportionality).
  - PCI-DSS Checklist: Controls like firewalls, encryption, unique IDs, logging.
  - Scripting Exercises: Simulated GRC tasks via bash (e.g., creating user registries, scanning vulnerabilities, generating compliance headers/reports).
